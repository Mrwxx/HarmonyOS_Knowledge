### （一）、应用跟随系统字体配置

1. **配置依赖**
   需在`AppScope/resources/base/profile/configuration.json`中启用跟随系统配置：

   ```
   {
     "configuration": {
       "fontSizeScale": "followSystem",  // 关键配置
       "fontSizeMaxScale": "3.2"         // 最大缩放倍数（可选）
     }
   }
   ```

   并在`AppScope/app.json5`中引用：2

   ```
   {
     "app": {
       "configuration": "$profile:configuration"  // 引用配置
     }
   }
   ```

### （一）、不同单位的字体区别

**绝对不要使用 `px` 作为文字单位**（`px` 是固定像素，系统字体放大时，文字不会同步自适应），ArkTS 中推荐使用以下两种相对单位：

- `vp`：视觉百分比单位，**适配屏幕尺寸 + 系统字体大小**，是文字尺寸的首选单位（鸿蒙官方推荐）。
- `fp`：字体专用相对单位，**仅适配系统字体大小**，不随屏幕尺寸变化，适合需要严格控制文字与其他元素比例的场景（如图标旁的说明文字）。



#### 1、fp

**fp单位特性**
`fp`是专为字体设计的逻辑像素单位（类似Android的sp）：1

- **默认规则**：`1fp = 1vp`
- **缩放机制**：当系统字体缩放比例为`scale`时，实际显示大小为 `fp值 × scale`
  （例如：`fontSize(16)`在系统缩放1.5倍时，实际显示为`24vp`）。
